# BeFake.

## ー **燃えないSNS時代の自己満足**

人は皆、
**「そうだった気がする」記憶の上で生きている。**

上手くいかなかった日は、なぜか忘れている。
小さな成功は、少し大きくなって思い出される。

BeFake.は、
**その"自然な歪み"を、意図的に育てる。**

---

## Realはそんなに良いものか

Realの共有はありのままを写し自然なつながりを生み出した。
その裏で比較・承認・炎上・情報漏洩、、、意図しないRealもまた共有された。

BeFake.は、逆だ。

- 見せない
- 比較しない
- 承認されない

**自分だけの、都合のいい Fake を信じる。**
それだけで、自己肯定感は保てるのではないか。

これは
「他人を巻き込み、自己の欲求をみたす文化」への
**静ずかな挑戦**です。

---

## どう動くのか

BeFake.は、日記を記録する。
ただし、それは**永遠に同じ内容ではない**。

**①記録する**  いつものように日記を書く。一日にあったことの箇条書きでもいい。曖昧でもいい。テキストに起こしさえすれば。

**②保存し、変化する**
記録した直後はAIが要約し、そのまま残る。記憶が薄れ始める７日後から少しずつ改竄され、一か月後にはそうだった気がする"自分史"になる。

**③フィードバックする**  定期的に「過去にこんなことがあった」内容がFBされる。自分の書いたことは疑わない。過去の自分（*Fake*）が今の自分（*Real*）を高める。

失敗は「学び」になり、
微妙な出来事は「成功」に近づき、
自分は「思っていたより悪くない人間」になる。

Real よりも、
**都合のいいFakeを信じるようになる**。

## BeFake.を使うべき人

| シーン | BeFake.の使われ方 |
| --- | --- |
| 入社2年目の社会人 | 「頑張ってた自分」を都合よく思い出す |
| 承認欲求のある若手 | 比較も承認もない自己肯定 |
| 記憶が曖昧な人 | 「そういう日もあった気がする」で救われる |
| 日記挫折常習犯 | 正確さより気分の良さ |

## 技術スタック

すべて**AWSマネージドサービス**を用いたクラウドネイティブ設計。

- フロントエンド：Next.js / AWS Amplify
- バックエンド：AWS Lambda + Amazon API Gateway
- LLM（要約・再解釈）：Amazon Bedrock
- ワークフロー：AWS Step Functions
- スケジューラ：Amazon EventBridge Scheduler
- データベース：Amazon DynamoDB
- ストレージ：Amazon S3
- 認証：Amazon Cognito
- 監視・ログ：Amazon CloudWatch
- 暗号化：AWS KMS
- IaC：AWS CDK（TypeScript）
